## Требования к среде / Environment Requirements

- devkitPro с devkitA64 и libnx
- Docker (для использования образа сборки)
- Python 3 (для waf/mpv)

## Сборка / Building

```bash
# С Docker
./build-docker.sh

# Вручную (требует devkitPro)
make configure
make libraries
make
```

## Стиль кода / Code Style

- Стандарт: C++23 (`-std=gnu++23`)
- Отступы: 4 пробела
- Комментарии только там, где логика неочевидна — НЕ описывайте что делает код, описывайте ПОЧЕМУ
- Используйте `SW_LOG_*` вместо `std::printf` для диагностических сообщений
- Предпочитайте `SW_LIKELY`/`SW_UNLIKELY` на горячих путях
- Для строк UI используйте `T::` константы из `src/i18n.hpp`

## Локализация / Localization

Все строки интерфейса находятся в:
- `src/i18n/ru.hpp` — русские строки (используются по умолчанию)
- `src/i18n/en.hpp` — английские строки

При добавлении новых строк обновляйте **оба** файла.

Для сборки с английским интерфейсом:
```
make CXXFLAGS="-DSWITCHWAVE_LANG=0"
```

## Логирование / Logging

| Макрос | Когда использовать |
|--------|-------------------|
| `SW_LOG_DEBUG` | Детали для отладки — в релизах не выводится |
| `SW_LOG_INFO`  | Нормальный ход работы (старт, инициализация) |
| `SW_LOG_WARN`  | Нежелательные, но восстановимые ситуации |
| `SW_LOG_ERR`   | Ошибки, требующие внимания |

Уровень по умолчанию: `INFO` в релизах, `DEBUG` при сборке с `-DDEBUG`.

## Pull Requests

1. Сделайте fork и создайте ветку от `main`
2. Коммиты должны быть атомарными с описательными сообщениями
3. Обновите `CHANGELOG.md` если изменение значимо для пользователей
4. Заполните шаблон PR
